# Global Instructions (Codex)

## Memory System (Highest Priority)

### Architecture

- `~/.codex/AGENTS.md`: global instructions, auto-loaded
- `~/.codex/lessons.md`: correction source-of-truth (append-only)
- `~/.codex/memory/codex_mem.db`: long-term observation store
- `~/.codex/MEMORY.md`: auto-loaded context (`model_instructions_file`), generated by `codex-mem`

### Session Startup Flow

Before the first substantive response in a session, refresh memory context:

```bash
~/.codex/bin/codex-mem build-context --project "$(pwd)" --output ~/.codex/MEMORY.md --lessons ~/.codex/lessons.md
```

### Self-Correction

**Identify corrections early**: user says something is wrong, says “remember / don't do this again”, expresses frustration, or same operation fails repeatedly.

**After a correction**:
1. Immediately write a lesson to `~/.codex/lessons.md` (date, context, mistake, rule)
2. Make the rule concrete and actionable
3. Refresh `~/.codex/MEMORY.md` using `codex-mem build-context`
4. Continue task execution only after recording
